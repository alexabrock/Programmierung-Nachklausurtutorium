\section{Hashing}

\section{Was ist Hashing?}
\begin{frame}{Was ist Hashing?}
  \begin{itemize}
    \item Hashing ist ein Verfahren, um Daten schnell zu finden.
    \item Es wandelt einen Schlüssel (z.B. einen String) anhand von einer Hashfunktion in einen Index um.
    \item Dieser Index zeigt auf eine Position in einem Hashtabellen-Array.
    \item Array-Platz `null` = leer 
    \item Ziel: contains() geht schnell
  \end{itemize}
\end{frame}

\section{Hashfunktionen}
\begin{frame}{Hashfunktionen}
  \begin{itemize}
    \item Eine Hashfunktion $h(k)$ nimmt einen Schlüssel $k$ und gibt einen Index zurück.
    \item Gute Hashfunktionen verteilen Schlüssel gleichmäßig auf die Hashtabellen-Plätze.
    \item Klassen definieren selbst `hashCode` Methode, die Hashfunktion definiert.
  \end{itemize}
\end{frame}

\section{Kollisionen}
\begin{frame}{Kollisionen}
  \begin{itemize}
    \item Kollision: Zwei Schlüssel werden auf denselben Platz abgebildet.
    \item Lösungsstrategien:
    \begin{itemize}
      \item Verkettung: Buckets sind Listen von Schlüsseln.
      \item Sondierung: Suche nach einem anderen freien Platz.
    \end{itemize}
  \end{itemize}
\end{frame}

\section{Hashset}
\begin{frame}{Hashset}
  \begin{itemize}
    \item Datenstruktur, die wie Menge funktioniert
    \begin{itemize}
      \item behält keine (sinnvolle) Sortierung
      \item keine doppelten Elemente
    \end{itemize}
    \item Grundidee: Objekte in großem Array speichern, Hashfunktion gibt Index vor
  \end{itemize}
\end{frame}

  Gegeben sei eine Hashtabelle der Größe 10 und die Hashfunktion $h(k) = k \mod 10$.

  \end{enumerate}
\end{frame}
